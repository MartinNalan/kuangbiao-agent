# Wireframes

## 1. 问答页面

```text
+----------------------------------------------------------+
| geowiki                                                  |
+----------------------------------------------------------+
| [基本模式 · 快速查证] [深度模式 · 综合研究]  本次 1/3 次 |
| 当前模式说明与证据边界                                   |
+----------------------------------------------------------+
| [ 输入专业问题，例如：哪个标准规定了金矿基本工程间距？ ] |
|                                             [发送]       |
+----------------------------------------------------------+
| 答案                                                     |
| 根据当前知识库检索结果，...                              |
|                                                          |
| 引用来源                                                 |
| 1. 文件名称 / 条款号 / 页码                              |
|    原文片段...                                           |
| 2. 文件名称 / 条款号 / 页码                              |
|    原文片段...                                           |
|                                                          |
| 检索依据                                                 |
| 全文搜索: 5 条 | 向量检索: 8 条 | 图谱查询: 2 条         |
|                                                          |
| [有用] [不准确] [复制答案]                               |
+----------------------------------------------------------+
```

深度模式处理中，答案位置显示：

```text
正在逐份检索候选文件                              57%
[==============================----------------------]
已审查 21/31 份                         证据覆盖 7 份
```

## 2. 页面元素

- 顶部标题
- 问题输入区
- 答案区
- 引用来源区
- 检索依据区
- 用户反馈区
- 会话历史入口
- 基本/深度模式分段选择和本次配额成本
- 深度任务阶段、进度、候选覆盖和刷新恢复

## 3. MVP 交互

1. 用户选择基本模式或深度模式。
2. 用户输入问题并点击发送。
3. 基本模式显示普通加载状态；深度模式创建持久化任务并显示阶段和进度。
4. 后端返回答案，或在深度任务完成后返回比较矩阵与覆盖范围。
5. 页面展示答案、短引文、官方链接和配额结算。
6. 基本答案适合扩大审查时，可一键转深度模式并只追加 2 次。
7. 用户点击反馈。

## 4. 管理员反馈队列

- 展示反馈时间、原问题、原因、补充说明、用户、处理通道和状态。
- `wrong_standard`、`wrong_clause`、`missing_evidence` 标记为知识库审核，不直接修改正式知识库。
- 管理员可更新为待处理、处理中、待 KB 审核、已解决或已忽略，并记录处理说明。
