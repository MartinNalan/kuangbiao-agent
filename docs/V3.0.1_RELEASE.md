# geowiki v3.0.1 Release Notes

## Technical Requirement Sufficiency

- Added the protected internal retrieval intent `technical_requirement_sufficiency` for questions such as whether completed research, tests, or higher-level work satisfies a stage requirement, covers a lower-level requirement, or still needs an additional step.
- Normalizes the nonstandard wording `实验室流程实验` to the standard term `实验室流程试验` without changing the business meaning.
- Retrieval now requires two independent evidence roles: the applicable stage requirement and a direct study-level relationship such as a test being performed on the basis of another test.
- Heading-only hits cannot satisfy this evidence gate. The answer must not invent that the user omitted a lower-level activity.
- Added prompt constraints for first semantic understanding, evidence review, and final answer generation; v3.0.1 verified the relation path with real local evidence from DZ/T 0205-2020 4.3.3 and DZ/T 0340-2020 5.2.2.

## Validation

- Added regression coverage for relation-intent preservation, terminology normalization, heading rejection, stage-plus-hierarchy evidence gating, source selection, and evidence-review activation.
- Full Python regression suite passes with 164 tests.
