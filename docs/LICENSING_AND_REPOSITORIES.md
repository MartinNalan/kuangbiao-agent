# Licensing and Repository Strategy

版本：v0.1
日期：2026-07-09
状态：草案

## 1. 目标

本项目采用“开源社区版 + 商业企业版”的双许可证路线。

目标：

- 允许个人和小团队免费本地部署、学习、研究和内部试用。
- 不在开源仓库分发真实标准全文、OCR 后标准全文或预构建真实标准知识库。
- 保留企业私有化部署、技术支持和增强功能的商业转化空间。
- 让社区版和企业版在代码、功能、许可证和数据边界上清晰分离。

## 2. 许可证策略

社区版建议采用：

```text
AGPL-3.0
```

企业版采用：

```text
Commercial License
```

含义：

- 社区版代码公开，适合个人本地部署、非商业研究和内部试用。
- 如果第三方基于社区版修改并作为网络服务提供，AGPL-3.0 通常要求其公开相应修改源码。
- 企业客户如需闭源集成、私有化部署、企业功能或商业支持，应使用商业许可证。
- 项目所有者自己拥有的代码，可以同时以 AGPL-3.0 和商业许可证授权。

注意：

- 如果外部贡献者参与社区版代码贡献，后续要将其贡献纳入企业版，建议引入 CLA 或明确贡献授权规则。
- 不应将无法商业再授权的第三方 AGPL 代码混入闭源企业版，除非符合其许可证要求。

## 3. 仓库规划

建议使用两个仓库：

```text
kuangbiao-agent          # 公开社区版，AGPL-3.0
kuangbiao-enterprise     # 私有企业版，Commercial License
```

### 3.1 社区版公开仓库

社区版可以包含：

- 基础前端。
- 基础后端。
- 本地部署脚本。
- 本地知识库 schema。
- 入库脚本。
- OCR 流程。
- 标准目录查询。
- 基础问答。
- API 文档。
- 架构文档。
- 示例/虚构数据。

社区版不得包含：

- 真实标准全文 PDF。
- OCR 后真实标准全文。
- 预构建真实标准知识库。
- 预构建真实标准向量库。
- 单位内部资料。
- 客户资料。
- API Key、token、`.env`。
- 商业版授权校验代码。
- 企业客户专属部署配置。

### 3.2 企业版私有仓库

企业版可以放在 GitHub 私有仓库，也可以不上传 GitHub，仅在本地或企业代码托管平台维护。

企业版可以包含：

- 企业用户和权限系统。
- SSO/LDAP/企业微信/飞书登录。
- 管理员审核后台。
- 审计日志。
- 组织级知识库。
- 高可用部署。
- 集群配置。
- 私有模型适配。
- 定制知识图谱。
- 商业授权校验。
- 企业安装脚本。
- 客户专属配置模板。

企业版仍不应包含未授权的标准全文公共库。客户私有资料应由客户自行提供或在授权范围内导入。

## 4. 数据边界

开源和商业代码可以分发，标准内容不能默认分发。

不得进入公开仓库：

- 真实标准 PDF。
- 标准 OCR 全文。
- 标准全文切块 JSON。
- 标准全文 embedding。
- 真实标准向量库。
- 真实标准全文 Elasticsearch dump。

可以进入公开仓库：

- 元数据字段定义。
- 空 schema。
- 虚构标准样例。
- 用户自建知识库流程。
- OCR 和入库脚本。
- 官方来源连接器代码。

## 5. 部署形态

### 5.1 社区本地版

用户自行 clone 仓库，本地部署，并导入自己合法持有的资料。

推荐流程：

```text
git clone <repo>
cp .env.example .env
docker compose up -d
put files into data/raw
run ingest
open local app
```

### 5.2 企业私有化版

企业客户购买商业许可证和部署服务，在企业内网或企业云环境中部署。

企业私有化版可按企业授权范围决定是否保存原始文件、OCR 全文和长文本引用。

### 5.3 云端服务版

云端服务版需要更严格的内容展示和权限控制。

未取得标准全文授权前：

- 不提供官方标准全文公共库。
- 不提供全文下载。
- 不提供大段原文展示。
- 用户上传资料默认私有。
- 公共知识库资料必须经过审核或授权。

## 6. GitHub 发布前检查清单

公开社区版发布前必须确认：

- `.env` 未提交。
- `data/`、`uploads/`、`models/`、`indexes/`、`*.pdf` 等真实资料未提交。
- 仓库包含许可证说明。
- README 明确不包含标准全文。
- 示例数据为虚构或公开许可数据。
- 文档说明用户需自行保证资料来源合法。

企业版发布或交付前必须确认：

- 使用商业许可证合同。
- 明确客户可用范围、部署范围、用户数和服务期限。
- 明确资料来源和标准全文授权责任。
- 明确是否允许保存 OCR 文本、向量库和索引。
