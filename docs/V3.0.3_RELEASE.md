# geowiki v3.0.3 Release Notes

发布日期：2026-07-16

## 修复范围

本版本修复“勘查阶段 + 矿石加工选冶技术性能要求”在基本模式和深度模式中的分类、检索、证据组织和渲染问题。

## 主要变更

- 新增受保护意图 `technical_stage_requirement`，不再将阶段性技术要求降级为通用问答。
- 建立 `DZ/T 0340-2020` 第 6.3、6.4、6.5 节与普查、详查、勘探阶段的章节关系，并自动展开对应的四个条件条款。
- 基本模式对条件未给全的问题返回完整条件矩阵，而不是要求用户先确认矿石类型；用户补充资源量规模和选冶难易程度后再定位唯一行。
- 深度研究规划器改为继承确定性原始意图和策略，修复模型规划结果默认变成 `cross_document_audit` 的问题。
- 阶段性技术要求深度模式采用单专题 `requirements_matrix`，只审查目标标准的直接条款。
- 修复非外推问题错误显示“工程间距、外推比例和适用条件”的固定摘要模板。

## 验证

- `PYTHONPATH=src .venv/bin/python -m unittest discover -s tests -q`
- 共 170 项自动化测试通过。
- 本地私有知识库验证：锂矿详查阶段选冶技术性能问题命中 `DZ/T 0340-2020` 第 6.4.1 至 6.4.4 条，并在基本和深度路径输出相同的条件矩阵。

## 数据边界

本版本没有同步或公开真实标准正文、私有知识库、向量索引、应用数据库、`.env` 或凭据。
