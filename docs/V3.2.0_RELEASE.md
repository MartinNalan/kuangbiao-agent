# geowiki v3.2.0

## Evidence Governance and Factual Grounding

- Positive-answer retrieval now shares one document governance predicate across FTS, KG, ANN, policy supplements, referenced tables and deep-research corpus enumeration.
- Only current, approved-for-service and answerable documents can support a business conclusion. Status lookup remains able to report repealed and unverified documents.
- Low-confidence model ambiguity now stops before retrieval and returns free-text clarification instead of searching with an unreliable intent.
- Generic terms such as `标准` and `报告` no longer admit unrelated questions into the mining-domain pipeline. Semantic `out_of_scope` classification also stops basic and deep execution before quota reservation.
- The output guard now validates supplied document identifiers and concrete claims, including values with units, dates, authority assertions and normative conditions. Unsupported drafts fall back to an evidence-only response.
- Catalog metadata no longer suppresses clause or table evidence from the same document. The `service_workflow` primary class also preserves the `service_time_limit` retrieval sub-strategy.
- When a user explicitly requests a type-level mining-right material overview, the system returns the governed Attachment 4 type overview; once a specific application type is selected, it returns only that type's Attachment 4 rows.
- `DZ/T 0430-2023` is currently `unverified` in the private KB and is therefore intentionally withheld from positive answers. KB task `T019` requires official status verification before it can be used again.

## Verification

- Added regression coverage for governance bypasses, expired-document exclusion, status-query exception, weak domain words, low-confidence ambiguity, semantic out-of-scope termination, fabricated Chinese document numbers, unsupported numeric claims, authority assertions and conditions.
